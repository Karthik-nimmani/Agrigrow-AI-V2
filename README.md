# AgriGrow AI

AgriGrow AI is a precision-farming decision-support platform built with
Next.js, Firebase, Genkit, and Gemini 2.5 Flash. The current application helps
farmers manage fields, synchronize weather information, ask agricultural
questions, and analyze crop images.

The project is being upgraded to implement the research framework described in
**Adaptive Agricultural Decision Intelligence (AADI): An Explainable Ensemble
Learning Framework for Robust Crop Yield Prediction**.

## Project Status

The repository currently contains a working Gemini-based application and the
research specification for the future AADI model.

### Available now

- Email/password, Google, and anonymous Firebase authentication
- User-owned farm fields stored in Cloud Firestore
- Real-time field and chat subscriptions
- Manual weather synchronization using Open-Meteo
- Reverse geocoding using Nominatim/OpenStreetMap
- Gemini-powered Agri-Bot
- Gemini-powered crop-image analysis
- Gemini-generated weather advice and field recommendations
- Responsive Next.js interface

### Prototype limitations

- The current yield number is generated by Gemini, not by a trained AADI model.
- Linear Regression, Random Forest, XGBoost, ensemble weighting, and SHAP are
  not yet implemented in the application.
- The phone OTP interface currently falls back to anonymous authentication.
- Soil-report extraction is simulated and is not a production OCR pipeline.
- Dashboard soil moisture is simulated.
- Dashboard potential yield uses a simple area-based calculation.
- Some inputs passed to the current Gemini forecast are hardcoded.
- No validated disease-detection accuracy percentage is claimed.

These limitations are documented deliberately so that research results and
prototype features are not confused.

## Research Objective

The AADI upgrade will create a reproducible crop-yield prediction pipeline that:

1. Learns from historical Indian agricultural and environmental data.
2. Predicts normalized crop yield in kilograms per hectare.
3. Trains and compares Linear Regression, Random Forest, and XGBoost.
4. Combines validated model predictions through a weighted ensemble.
5. Tests geographic generalization using district-grouped cross-validation.
6. Evaluates performance using R2, RMSE, MAE, and SMAPE.
7. Explains predictions using model coefficients and SHAP values.
8. Serves predictions through a versioned Python API.
9. Uses Gemini only to convert fixed model results into farmer-friendly advice.

The detailed research contract is available in
[`docs/research-model-spec.md`](docs/research-model-spec.md).

## Intended Final Architecture

```text
Next.js farmer interface
        |
        +--> Firebase Authentication and Firestore
        |
        +--> Python FastAPI model service
                  |
                  +--> Validated preprocessing pipeline
                  +--> Linear Regression
                  +--> Random Forest
                  +--> XGBoost
                  +--> Weighted ensemble
                  +--> SHAP explanations
        |
        +--> Gemini farmer-friendly explanation
```

The AADI model will calculate the numerical yield. SHAP will explain the model
mathematically. Gemini will communicate the unchanged result in simple language.

## Current Application Data Flow

1. A user signs in through Firebase Authentication.
2. The app creates or updates the profile under `/users/{userId}`.
3. Fields are stored under `/users/{userId}/farmFields/{fieldId}`.
4. Firestore snapshot listeners update the UI in real time.
5. Manual weather sync obtains browser coordinates and calls Open-Meteo.
6. Current field, weather, image, or question data is sent to a Genkit flow.
7. Gemini returns a structured response validated with Zod.
8. Chat messages are stored under `/users/{userId}/chatMessages/{messageId}`.

## Development Phases

1. **Research foundation - complete:** target, features, leakage rules, models,
   metrics, validation, explainability, and API contract defined.
2. **Data pipeline:** acquire, inspect, clean, normalize, join, and test datasets.
3. **Baseline models:** train and compare LR, Random Forest, and XGBoost.
4. **Ensemble validation:** derive weights and run spatial cross-validation.
5. **Explainability:** add SHAP and generate paper-style evaluation reports.
6. **Model API:** serve the saved pipeline using FastAPI.
7. **Application integration:** connect Next.js and store forecast history.
8. **Hybrid AI and testing:** retain Gemini features and verify end-to-end behavior.

Each phase is discussed first, implemented only after approval, and tested before
the project moves to the next phase.

## Local Setup

### Requirements

- Node.js 20 or later
- npm
- Firebase project with Authentication and Firestore configured
- Gemini API key for current generative features

### Install and run

```bash
npm install
npm run dev
```

The application runs at [http://localhost:9002](http://localhost:9002).

### Environment

Add a supported Gemini key to `.env`:

```text
GOOGLE_GENAI_API_KEY=your_key
```

`GEMINI_API_KEY` and `GOOGLE_API_KEY` are also recognized by the current Genkit
configuration. Do not commit private keys.

### Checks

```bash
npm run typecheck
npm run build
```

## Research Integrity

- Do not describe Gemini output as a trained ensemble prediction.
- Do not describe R2 as an accuracy percentage.
- Do not use production or post-harvest information as prediction features.
- Do not publish a metric unless it can be reproduced from versioned code and
  traceable data.
- Record preprocessing, model versions, ensemble weights, and evaluation splits.
